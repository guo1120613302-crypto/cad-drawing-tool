# CAD 图层与工具改进需求文档

## 1. 概述

本需求旨在改进 CAD 应用程序中的图层管理和工具功能，解决图层属性丢失、删除提示不合理、缺少撤销/重做功能以及工具适用范围受限等问题。

## 2. 用户故事

### 2.1 图层属性保持
**作为** CAD 用户  
**我希望** 使用任何工具（偏移、修剪、延伸、打断、旋转、镜像、移动等）处理图形后，新生成的图形能够继承原图形的图层属性  
**以便** 保持图层组织的一致性，不需要手动重新分配图层

### 2.2 图层删除提示优化
**作为** CAD 用户  
**我希望** 只在删除基准图层"0"时收到警告提示，删除其他图层时直接删除  
**以便** 提高工作效率，减少不必要的确认步骤

### 2.3 撤销/重做功能
**作为** CAD 用户  
**我希望** 使用 Ctrl+Z 撤销操作和 Ctrl+Y（或 Ctrl+Shift+Z）重做操作  
**以便** 能够恢复误操作，包括恢复已删除的图层和图形

### 2.4 工具通用性增强
**作为** CAD 用户  
**我希望** 延伸和打断工具能够作用于任何图形的边（包括矩形、多边形等），而不仅限于直线  
**以便** 更灵活地编辑各种类型的图形

## 3. 功能需求

### 3.1 图层属性继承机制

#### 3.1.1 工具生成图形的图层继承
- 所有编辑工具（偏移、修剪、延伸、打断、旋转、镜像、移动）在生成新图形时，必须继承源图形的图层属性
- 图层属性包括：
  - 图层名称 (layer_name)
  - 图层颜色
  - 可见性 (visible)
  - 锁定状态 (locked)
  - 不透明度 (opacity)

#### 3.1.2 实现方式
- 在每个工具的 `_finalize_*` 或命令执行方法中，调用 `layer_manager.apply_layer_props_from_source(new_item, source_item)` 方法
- 该方法应从源图形读取图层属性并应用到新图形

### 3.2 图层删除提示优化

#### 3.2.1 基准图层保护
- 删除图层名称为"0"的基准图层时，显示警告对话框
- 警告内容：提示这是基准图层，询问是否确认删除

#### 3.2.2 普通图层删除
- 删除非基准图层时，不显示确认对话框
- 直接删除图层及其上的所有图形
- 删除操作应支持撤销

### 3.3 撤销/重做功能增强

#### 3.3.1 图层删除的撤销支持
- 创建 `CommandDeleteLayer` 撤销命令类
- 记录被删除图层的所有属性和图形
- 撤销时恢复图层定义和所有图形

#### 3.3.2 快捷键支持
- Ctrl+Z：撤销上一步操作
- Ctrl+Y 或 Ctrl+Shift+Z：重做已撤销的操作
- 快捷键应在 `keyPressEvent` 中正确处理

### 3.4 延伸工具通用性

#### 3.4.1 支持多边形边的延伸
- 延伸工具应能识别多边形的各条边
- 用户点击多边形的某条边时，该边应能延伸至最近的边界
- 延伸后，多边形应更新为新的坐标集

#### 3.4.2 边界检测增强
- 射线碰撞检测应考虑所有类型图形的边
- 包括：直线、矩形的四条边、多边形的所有边

### 3.5 打断工具通用性

#### 3.5.1 支持多边形边的打断
- 打断工具应能识别多边形的各条边
- 用户点击多边形的某条边时，该边应在点击位置断开
- 打断后，原多边形应分解为多条独立的线段

#### 3.5.2 投影点计算
- 对于多边形，应找到离鼠标最近的边
- 计算鼠标在该边上的投影点作为打断点

## 4. 验收标准

### 4.1 图层属性继承
- [ ] 使用偏移工具生成的图形继承源图形的图层属性
- [ ] 使用修剪工具生成的图形继承源图形的图层属性
- [ ] 使用延伸工具生成的图形继承源图形的图层属性
- [ ] 使用打断工具生成的图形继承源图形的图层属性
- [ ] 使用旋转工具生成的图形继承源图形的图层属性
- [ ] 使用镜像工具生成的图形继承源图形的图层属性
- [ ] 使用移动工具移动的图形保持原图层属性

### 4.2 图层删除提示
- [ ] 删除基准图层"0"时显示警告对话框
- [ ] 删除其他图层时不显示确认对话框，直接删除
- [ ] 删除图层后，该图层上的所有图形也被删除

### 4.3 撤销/重做功能
- [ ] Ctrl+Z 能够撤销图层删除操作
- [ ] Ctrl+Z 撤销图层删除后，图层定义和图形都恢复
- [ ] Ctrl+Y 能够重做已撤销的图层删除操作
- [ ] 撤销/重做功能不影响其他操作的撤销栈

### 4.4 延伸工具通用性
- [ ] 延伸工具能够延伸直线
- [ ] 延伸工具能够延伸矩形的边
- [ ] 延伸工具能够延伸多边形的边
- [ ] 延伸预览正确显示将要延伸的边

### 4.5 打断工具通用性
- [ ] 打断工具能够打断直线
- [ ] 打断工具能够打断矩形的边
- [ ] 打断工具能够打断多边形的边
- [ ] 打断预览正确显示打断点位置

## 5. 非功能需求

### 5.1 性能
- 图层属性继承操作应在 10ms 内完成
- 撤销/重做操作应在 50ms 内完成

### 5.2 兼容性
- 所有修改应与现有代码架构兼容
- 不应破坏现有的撤销栈机制

### 5.3 用户体验
- 操作应流畅，无明显卡顿
- 错误提示应清晰明确

## 6. 技术约束

- 使用 PyQt6 框架
- 遵循现有的 QUndoCommand 撤销栈架构
- 使用 GeometryEngine 进行几何计算
- 保持 SmartLineItem 和 SmartPolygonItem 的接口一致性

## 7. 优先级

1. **高优先级**：图层属性继承机制（问题 1）
2. **中优先级**：图层删除提示优化（问题 2）、撤销/重做功能（问题 3）
3. **低优先级**：工具通用性增强（问题 4）

## 8. 风险与依赖

### 8.1 风险
- 修改撤销栈可能影响现有功能的稳定性
- 工具通用性增强可能引入新的边界情况

### 8.2 依赖
- 依赖 GeometryEngine 的几何计算能力
- 依赖 LayerManager 的图层管理功能
- 依赖 QUndoStack 的撤销/重做机制
