# CAD 图层与工具 Bug 修复总结

## 修复日期
2026-03-13

## 修复的问题

### 1. ✅ 图层属性继承问题
**问题描述**：使用工具（偏移、修剪、延伸、打断、旋转、镜像、移动）处理图形后，新生成的图形不再属于原图层。

**修复内容**：
- 在 `LayerManager` 中添加了 `copy_layer_props(target_item, source_item)` 方法
- 修复了以下工具的图层继承：
  - ✅ `tool_offset.py` - 偏移工具
  - ✅ `tool_trim.py` - 修剪工具
  - ✅ `tool_break.py` - 打断工具
  - ✅ `tool_extend.py` - 延伸工具（保存图层信息到撤销命令）
  - ✅ `tool_mirror.py` - 镜像工具
  - ✅ `tool_rotate.py` - 旋转工具（保持图层属性）
  - ✅ `tool_move.py` - 移动工具（保持图层属性）

**技术细节**：
- 新建图形的工具（offset, trim, break, mirror）：调用 `copy_layer_props()` 从源图形复制图层属性
- 修改现有图形的工具（extend, rotate, move）：在撤销命令中保存和恢复 `layer_name` 属性

### 2. ✅ 图层删除提示优化
**问题描述**：删除任何图层都显示确认对话框，应该只对基准图层"0"显示警告。

**修复内容**：
- 修改 `LayerManager.delete_layer()` 方法
- 只对基准图层"0"显示确认对话框
- 其他图层直接删除，无需确认
- 删除操作支持撤销/重做

**技术细节**：
- 创建了 `CommandDeleteLayer` 撤销命令类
- 保存被删除图层的所有属性和图形
- 撤销时完整恢复图层定义和图形

### 3. ✅ 撤销/重做功能增强
**问题描述**：需要 Ctrl+Z 撤销和 Ctrl+Y 重做功能，能够恢复图层。

**修复内容**：
- 图层删除操作已集成到撤销栈
- 使用 `QUndoCommand` 封装图层删除操作
- 支持完整的图层恢复（包括图层属性和所有图形）

**已有功能**：
- Ctrl+Z 撤销 - 已在 `core_canvas.py` 中实现
- Ctrl+Y 重做 - 已在 `core_canvas.py` 中实现
- 所有工具操作都已支持撤销/重做

### 4. ⚠️ 工具通用性增强（待实现）
**问题描述**：延伸和打断工具目前只能作用于直线，应该能作用于任何图形的边。

**当前状态**：
- 打断工具已支持多边形边的打断
- 延伸工具目前只支持直线延伸

**需要进一步实现**：
- 延伸工具支持多边形边的延伸
- 需要修改 `tool_extend.py` 中的 `_get_extend_data()` 方法
